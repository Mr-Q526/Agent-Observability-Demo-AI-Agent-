Agent Pipeline 执行流程

🔀 三种路由模式

NLU 意图识别后，根据意图类型路由到不同复杂度的 Pipeline。越复杂的任务经过越多阶段。

● Intent 模式

简单意图（如问候、闲聊）。跳过检索和计划，直接由 LLM ReAct 循环处理。

Guard → NLU → ReAct → Guard

● Skill 模式

需要工具的任务（如搜索商品、查订单）。增加自动检索和反思阶段。

Guard → NLU → RAG → ReAct → Reflect → Guard

● Workflow 模式

多步确定性任务（如批量商品对比）。代码驱动 for 循环，LLM 仅参与总结。

Guard → NLU → RAG → Observe → Plan → Workflow → Reflect → Guard
📋 各阶段详细说明
🛡️ ① Input Guardrails (输入安全护栏) 每次触发
用户消息进入后的第一道防线。支持 Regex 和 LLM 两种检查模式，检测注入攻击、敏感词等。
若检测不通过，直接拦截并返回拒绝回复，不进入后续流程。
触发：每次用户发送消息
🧠 ② NLU 意图路由 + Slot 抽取 每次触发
分析用户输入的意图和关键参数。支持 Embedding 向量匹配和 LLM 推理两种路由方式（可在面板切换）。
识别出意图（如 search_product、refund）、Slot 实体（如 order_id=ORD001）、缺失的必填 Slot。
同时确定路由类型：intent / skill / workflow，决定后续经过哪些阶段。
触发：每次用户发送消息（安全检查通过后）
📚 ③ Auto-Retrieve (自动检索) Skill / Workflow
根据意图自动选择合适的检索集合（知识库 / 商品库），使用 Hybrid RAG（关键词 + 向量 + RRF 融合）检索相关上下文。
检索到的信息会注入到 System Prompt 中，让 LLM 基于真实数据回复。
触发：意图为 search_product / knowledge_query 等需要外部信息的 skill 或 workflow
👁️ ④ Observe (观察) 仅 Workflow
整合 NLU 结果、检索上下文和用户记忆，生成结构化的观察报告。
为 Plan 阶段提供完整的上下文信息。
触发：路由类型为 workflow 时
🗺️ ⑤ Plan (计划) 仅 Workflow
读取技能定义中 预定义的结构化执行步骤（代码驱动，确定性）。
每步定义了 action (tool/llm)、要调用的工具名称和 Slot 映射。与 ReAct 模式的根本区别：流程由代码决定，不是 LLM 动态决策。
触发：路由类型为 workflow 或 skill 时
🧩 ⑥ Prompt Assembly (Prompt 组装) 每次触发
将 Base Prompt、技能专属规则、记忆上下文、缺失 Slot 追问指示、RAG 检索结果、执行计划等多层信息拼装为最终 System Prompt。
同时记录 Memory 四层快照（工作记忆、任务记忆、用户画像、长期摘要）。
触发：每次调用 LLM 前
🔄 ⑦ Act: ReAct 循环 / Workflow 执行 核心
ReAct 模式（intent / skill）：LLM 自主决定调用哪些工具，每轮产生 Thought → Action → Observation，最多循环 5 次。
Workflow 模式：代码 for 循环按计划依次执行工具调用，LLM 仅在最后一步生成总结回复。
遇到敏感操作（如退款）会暂停等待用户确认。
触发：Pipeline 的核心执行阶段
🪞 ⑧ Reflect (反思) Skill / Workflow
检查执行结果的质量：回复是否过短、工具是否出错、是否需要重试。
质量等级：good / warning / poor。
触发：有执行计划的 skill 或 workflow 执行完成后
🛡️ ⑨ Output Guardrails (输出安全护栏) 每次触发
对 LLM 生成的最终回复进行安全检查，包括幻觉检测（对比工具返回的真实数据）和敏感内容过滤。
若发现虚构数据，自动用真实数据修正回复。
触发：LLM 生成回复后、返回给用户前
📝 ⑩ Memory Note Update (记忆笔记更新) 每次触发
调用 LLM 滚动更新记忆笔记（rolling summary）。保留用户偏好、推荐过的商品、操作结果等关键信息，控制在 100 字内。
笔记会注入到下一轮的 System Prompt 中，实现跨轮记忆。
触发：每轮对话完成后
